# Experiment 04 — repeat-train-seed stability

This experiment is the hard gate before formal topology-only GNN training. It
separates the effect of the 40/10 training/validation draw from the topology and
from the fixed evaluation population.

## Locked protocol

- Select 60 topologies deterministically from the completed Experiment 03 labels.
- Keep the Experiment 03 seed-42 label as the reference observation.
- Generate fresh complete `(X, y)` train/validation samples for train seeds 43
  and 44: 40 training and 10 validation samples per topology and seed.
- Reuse the exact Experiment 03 1,000-sample test NPZ and hash for every repeat.
  A new test bank is never generated by this experiment.
- Run 2stage and SPO+ with `max_epochs=10000`, patience 20, min delta 0.0001,
  and require both methods to stop early.
- New work is 60 topologies × 2 train seeds = 120 paired jobs. The final audit
  contains 180 labels after joining the 60 Experiment 03 seed-42 observations.

The selection has 10 each of ceiling-zero, nonzero-gap plateau, small-nonzero,
material-positive, and material-negative cases, plus five extreme-positive and
five extreme-negative cases. Selection is deterministic across complexity
quantiles within each category.

## Commands

From the repository root:

```bash
python surrogate_experiment_results/Step5_topology_dfl_suitability/experiment_04_repeat_seed_stability_sample50/scripts/select_repeat_seed_topologies.py

python surrogate_experiment_results/Step5_topology_dfl_suitability/experiment_04_repeat_seed_stability_sample50/scripts/build_repeat_seed_artifacts.py --dry-run

python surrogate_experiment_results/Step5_topology_dfl_suitability/experiment_04_repeat_seed_stability_sample50/scripts/audit_repeat_seed_artifacts.py

python surrogate_experiment_results/Step5_topology_dfl_suitability/experiment_04_repeat_seed_stability_sample50/scripts/plan_repeat_seed_jobs.py
```

Launcher preview and execution reuse the audited Experiment 01 bounded launcher:

```bash
python surrogate_experiment_results/Step5_topology_dfl_suitability/experiment_01_weak_label_seed42_sample50/scripts/launch_weak_label_jobs.py \
  --jobs-csv surrogate_experiment_results/Step5_topology_dfl_suitability/experiment_04_repeat_seed_stability_sample50/results/repeat_seed120/plans/repeat_seed120_jobs.csv \
  --output-root surrogate_experiment_results/Step5_topology_dfl_suitability/experiment_04_repeat_seed_stability_sample50/results/repeat_seed120 \
  --expected-job-count 120 --normal-workers 20 --long-workers 0 --require-hostname garnet
```

Add `--execute` only on Garnet after the artifact audit and preview pass. Formal
GNN training remains disabled until `review_repeat_seed_results.py` reports both
integrity success and the documented stability gate result.

The Garnet-only wrapper `scripts/run_garnet_pipeline.sh` performs that exact
gated sequence in one resumable pipeline: eight artifact shards, strict audit,
strict 120-row plan, launcher preview, 20-worker execution, then final review.
It never invokes Experiment 05 training.

## Outputs

- `configs/topologies.repeat60.csv`: locked 60-topology panel.
- `results/repeat_seed120/results/repeat_seed_artifact_audit.json`: explicit
  fixed-test-bank and fresh-train-seed audit.
- `results/repeat_seed120/results/repeat_seed_labels_long.csv`: 180 seed-level labels.
- `results/repeat_seed120/results/repeat_seed_stability_summary.csv`: one row per topology.
- `results/repeat_seed120/results/repeat_seed_review_audit.json`: correlations,
  plateau/sign consistency, and the GNN gate decision.
